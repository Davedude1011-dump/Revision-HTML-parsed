These registers play a vital role in managing various aspects of the CPU's performance and interaction with other system components.</div> <div class="header">1. Program Counter (PC)</div> <div class="text">- **Function:** The Program Counter holds the memory address of the next instruction to be fetched and executed. It ensures the sequential execution of program instructions.</div> <div class="text">- **Role:** Keeps track of the instruction pointer, advancing to the next instruction after each execution cycle.</div> <div class="header">2. Instruction Register (IR)</div> <div class="text">- **Function:** The Instruction Register holds the current instruction fetched from memory. It allows the CPU to decode and execute the instruction.</div> <div class="text">- **Role:** Facilitates the communication of instructions between the CPU and memory, enabling the execution of machine code.</div> <div class="header">3. Memory Address Register (MAR)</div> <div class="text">- **Function:** The Memory Address Register stores the memory address for read or write operations. It indicates the location in memory where data is to be fetched or stored.</div> <div class="text">- **Role:** Facilitates communication between the CPU and memory, ensuring data is accessed from the correct memory location.</div> <div class="header">4. Memory Buffer Register (MBR)</div> <div class="text">- **Function:** The Memory Buffer Register temporarily holds data either fetched from memory or data to be written to memory.</div> <div class="text">- **Role:** Acts as an intermediary between the CPU and memory, allowing the transfer of data during read or write operations.</div> <div class="header">5. Status Register (SR or Flags Register)</div> <div class="text">- **Function:** The Status Register contains various flags that indicate the current state of the processor, such as zero flag, carry flag, and overflow flag.</div> <div class="text">- **Role:** Enables the CPU to make decisions based on the outcome of arithmetic and logical operations, influencing program flow.</div> <div class="header">6. Stack Pointer (SP)</div> <div class="text">- **Function:** The Stack Pointer keeps track of the current top of the stack in memory. It is crucial for managing the program's execution flow, especially during subroutine calls and returns.</div> <div class="text">- **Role:** Facilitates the implementation of the call stack, allowing the CPU to manage function calls and store return addresses.</div> <div class="header">7. Input/Output Registers (I/O Registers)</div> <div class="text">- **Function:** I/O Registers act as interfaces between the CPU and peripheral devices. They store data related to input or output operations.</div> <div class="text">- **Role:** Enables the CPU to communicate with external devices, facilitating data transfer and control signals.</div> <div class="header">Conclusion</div> <div class="text">Special Purpose Registers are integral to the functioning of a CPU, providing essential functionalities for executing instructions, managing memory, and interacting with external devices.
